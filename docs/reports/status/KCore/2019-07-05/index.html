KCore Status Report (2019-07-05)

The "KCore" team is the team responsible for the core server and hosting infrastructure in ASP.NET Core, as well as ASP.NET Core SignalR and some Microsoft.Extensions components.

Overview

Preview 7 is code-complete! Some additional verification is still underway but we're mostly ready for release.

External contributions during these two weeks:

  • Nick Darvey contributed a fix to SignalR to properly support derived-type arguments in Hub methods.
  • Ben Adams did the perf. Always doing the perf, he is.
  • Aristarkh Zagorodnikov taught our "systemd" support how to detect older versions of systemd.
  • huysentruitw improved how tests using TestServer can access DI services by getting access to the DI container from the RequestBuilder.

The team was hard at work, as always:

  • We dramatically improved the performance of the SignalR JSON protocol thanks to new APIs in System.Text.Json. A 2x improvement in performance!
  • We improved the performance of SignalR's response to the /negotiate request.
  • We are changing the package name for the SignalR JavaScript client from @aspnet/signalr to @microsoft/signalr.
  • We improved Kestrel's client certificate configuration to better support client-certificate authentication.
  • SignalR now uses the new "Bedrock" client abstractions, unifying more of the client and server APIs between Kestrel and SignalR and creating examples on which future .NET-based network apps will be written.
  • We fixed a major performance regression in Kestrel.
  • We converted TestServer to use System.IO.Pipelines natively.
  • We removed a redundant call to AuthenticateAsync in IIS out-of-proc, which also removed problematic side-effects.
  • We consolidated configuration of "Windows Auth" across both the native support in HttpSys/IIS and the new cross-platform support in 3.0.
  • We fixed a major HTTP/2 race condition in Kestrel which was exposed in recent builds.
  • We converted the ASP.NET Core Runtime Site Extension back to use semver v1 to work-around a bug in Azure App Services.
  • We suppressed an unhelpful and unnecessary error message caused by users attempting to "complete" the Request body pipe.
  • So. Many. Doc comments.
  • We converted the Microsoft.Extensions.Caching.SqlServer component to use the new Microsoft.Data.SqlClient library.
  • The Request Queuing middleware now has useful Event Counters and and EventSource.
  • The Request Queuing middleware has a pluggable queuing policy as well as a new Last In, First Out policy.
