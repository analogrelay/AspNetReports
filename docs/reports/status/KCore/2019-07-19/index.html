KCore Status Report (2019-07-19)

The "KCore" team is the team responsible for the core server and hosting infrastructure in ASP.NET Core, as well as ASP.NET Core SignalR and some Microsoft.Extensions components.

Overview

We are deep in to Preview 8 work, wrapping up the last few things for 3.0. Stress is the name of the game, but hopefully just stressing the product and not the engineers ;). We're diving in to the reliability of Kestrel's HTTP/2 stack under heavy load thanks to some tools we're sharing with the CoreFX team. Other areas of the stack are seeing some final API tweaks to get things ready for 3.0. Of course, we're continuing to knock down the last few bugs as well. We also got some help from Maryam on the CoreFX team with some Extensions work items we're trying to knock down for preview 8!

External Contributions this week:

  • Stafford Williams tidied up one of our SignalR benchmarking tools.
  • TeBeCo fixed a synchronous I/O usage in one of our SignalR samples.
  • Carlos J. Aliaga fixed an invalid cast when working with custom header values.
  • Ben Adams made more methods inlinable by wrapping 'throw's in methods.
  • Daniel Martin added some useful new events for JWT Bearer Authentication.

The team continues to work on getting a quality product ready for release:

  • We added some logging to help users avoid a hang when IHost isn't disposed.
  • We fixed a small memory leak in the SignalR Java Client.
  • We fixed a race in the SignalR Java Client.
  • We got the SignalR Endpoint Routing APIs aligned with other Endpoint Routing APIs.
  • We removed the use of the C++ Concurrency Runtime (PPL) in our prototype SignalR C++ client.
  • The concurrency limit middleware, coming after 3.0, got some polish.
    • Dylan, our intern, is getting close to finishing his internship, he finishes on August 2nd!
    • Check him out on Twitter: @DylanDmitriGray.
  • Lots of Kestrel perf stuff: #12265, #12225, #12285.
  • We quieted down a noisy client-triggered log message. This was a long-standing customer request!
  • Lots of work on Kestrel stress-resiliance: #12081, #12204, #12131,
  • We started building our own packages for VS insertion of ASP.NET Core components in 2.1/2.2.
  • We updated to the latest version of Microsoft.IdentityModel and friends.
